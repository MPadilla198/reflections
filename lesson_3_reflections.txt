When would you want to use a remote repository rather than keeping all your work local?

When collaborating with others, when saving progress on the cloud to have access to the code on any other computer.


Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?

By pulling manually, the developer has the option of choosing when to showcase a new feature to others.


Describe the differences between forks, clones, and branches. When would you use one instead of another?

A fork is a variation of someone else's project on GitHub. A clone is a project on GitHub being brought to local as a repository. A branch is a variation in the code.


What is the benefit of having a copy of the last known state of the remote stored locally?

To be able to modify the most recent version of the application.
